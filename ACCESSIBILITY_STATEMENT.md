# Accessibility Statement

**Last Updated**: January 2026

## Commitment to Accessibility

TanStack Wonderland is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards.

## Standards

We aim to conform to the **Web Content Accessibility Guidelines (WCAG) 2.2 Level AA** standards. These guidelines explain how to make web content more accessible for people with disabilities.

## Accessibility Features

Our application includes the following accessibility features:

### Keyboard Navigation
- All interactive elements are keyboard accessible
- Logical tab order throughout the application
- Keyboard shortcuts for common actions
- Focus indicators visible on all interactive elements

### Screen Reader Support
- Semantic HTML structure
- ARIA labels and roles where appropriate
- Alt text for images
- Descriptive link text
- Form labels associated with inputs

### Visual Accessibility
- Sufficient color contrast ratios (WCAG AA compliant)
- Text resizable up to 200% without loss of functionality
- No reliance on color alone to convey information
- Dark mode support for reduced eye strain

### Responsive Design
- Works on various screen sizes and devices
- Touch-friendly interface elements (minimum 44x44px)
- Mobile-optimized layouts

## Known Issues

We are aware of the following accessibility issues and are working to address them:

1. **Inline Styles**: Some dynamic progress bars use inline styles (necessary for functionality)
2. **Complex Tables**: Large data tables may be challenging for screen readers (virtualization required for performance)

## Testing

We test our application for accessibility using:
- Automated testing tools (axe-core)
- Manual keyboard navigation testing
- Screen reader testing (NVDA, JAWS, VoiceOver)
- Color contrast analyzers

## Continuous Improvement

We are committed to continuously improving accessibility. Our development process includes:
- Accessibility reviews in code reviews
- Regular accessibility audits
- User feedback integration
- Training for developers on accessibility best practices

## Third-Party Content

Some content on our site is provided by third parties (e.g., PokeAPI). We cannot guarantee the accessibility of third-party content, but we work with providers to ensure accessibility standards are met.
